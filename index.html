The Power of Choice

Welcome back, Joseph. Until now, your code ran in a straight line. Like a DVD logo bouncing off the walls, it had no intelligence. Conditionals give your code a brain. They allow your program to look at the world and say: "If this happens, I will do this. Otherwise, I will do that."

1. Booleans (bool)

Why learn this?

Think of a video game. How does the game know if you have the key to the boss door? It uses a Boolean. Computers are secretly just billions of microscopic light switches. A Boolean is exactly that: a switch that is strictly True (ON) or False (OFF). Notice the capital letters!

We create Booleans by asking Python questions using Comparison Operators:

  • > Greater than
  • < Less than
  • >= Greater or equal
  • <= Less or equal
  • == Exactly equal to
  • != NOT equal to

The Ultimate Trap: = vs ==

Beginners mess this up daily.
= (One equals) is a command: "Put this sword in my inventory." (weapon = "sword")

== (Two equals) is a question: "Do I have the sword?" (weapon == "sword")

2. The Crossroads (if, elif, else)

Why learn this?

Imagine walking through a dungeon. You see a giant door. if you have the boss key, you enter. elif (Else If) you have a lockpick, you sneak through the side door. else (if you have nothing), you fall into a trapdoor. This is how games make choices based on your stats!

health = int(input("What is your HP? "))

if health == 100:
    print("You are at max health! Ready for battle.")
elif health >= 50:
    print("You are wounded, but can still fight.")
elif health > 0:
    print("WARNING: Drink a health potion immediately!")
else:
    print("GAME OVER")

The Order Trap

Python is a loyal robot. It reads your code top-to-bottom. The VERY FIRST condition that is True wins. Everything else below it is completely ignored.

If you put elif health > 0 at the very top, and the player had 100 health... Python would say "Yep, 100 is greater than 0!" and print the WARNING message, ignoring the max health message entirely. Always put your hardest-to-reach or most specific condition FIRST!

3. Logic Gates (and, or)

Why learn this?

Sometimes life is complicated. You can only play Xbox if your homework is done and your room is clean. Or, maybe you can win a level by defeating the boss or by finding the secret exit. We use these words to glue multiple Booleans together.

and

STRICT: Both sides MUST be True. If even one is False, the whole thing fails.

# Needs both key AND high level
if has_key and level >= 10:
    print("Chest opened!")

or

FORGIVING: Only ONE side needs to be True. It doesn't care about the other.

# Either money OR intimidation works
if gold > 100 or strength > 50:
    print("Guard lets you pass.")

4. The Matchmaker (match)

Why learn this?

Because programmers are delightfully lazy! Imagine a character selection screen with 20 fighters. Writing elif name == "Sub-Zero" 20 times is exhausting. In Python 3.10, they added match. It acts like a vending machine: you drop a variable in, and it finds the matching slot instantly.

character = input("Choose your fighter: ")

match character:
    case "Mario" | "Luigi": # The | symbol means 'or' here!
        print("You chose a plumber!")
    case "Bowser":
        print("You chose the boss!")
    case _: # This underscore acts exactly like 'else'
        print("Fighter locked.")
